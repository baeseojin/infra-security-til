## 📝 오늘 배운 내용 요약

1. OSI 모델
    - osi모델에 각 계층은 헤더와 data/payload로 정의
    - 송신 측이 헤더를 생성하여 추가하면 수신 측에서 해당 계층이 이 헤더를 사용
    - 상위 계층이나 하위 계층 사이에 주고 받는 것을 서비스 데이터 단위(sdu)
    - 같은 계층 사이에서 주고 받는 것을 프로토콜 데이터 단위(pdu)
    - **송신 호스트(클라이언트)** - 상위에서 하위 계층으로 내려보내면서 계속 자기 계층 헤더를 붙인다.
        - 캡슐화
        - 계층 건너 띄기 안됨
    - **수신 호스트(서버)** - 하위에서 상위 계층으로 올려 보내면서 계속 자기 계층 헤더를 뗀다.
        - 역캡슐화
        - 계층 건너 띄기 안됨
    - 프로토콜
        - 데이터 단위
            - ADPU : 응용 계층의 데이터 단위 (7)
            - PPDU : 표현 계층의 데이터 단위 (6)
            - SPDU : 세션 계층의 데이터 단위 (5)
            - TDPU : 전송 계층의 데이터 단위 (4)
                - 세그먼트 : TCP 프로토콜에서 사용
                - 데이터그램 : UDP 프로토콜에서 사용
            - NPDU : 네트워크 계층의 데이터 단위 (3)
                - 패킷
            - DPDU : 데이터 링크 계층의 데이터 단위 (2)
                - 프레임
            - PPDU : 물리 계층의 데이터 단위 (1)
                - 비트
    - 2 계층에 꼬리말인 **트레일러** 존재
        - 오류 검증을 함(송신,수신 모두 존재)
2. OSI 모델에 각 계층에 대한 설명
    1. <물리 계층>
        1. 두 시스템 간에 데이터를 전송하려고 링크를 활성화해 관리하는 **전기적, 기계적**, 절차적, 기능적 특성 등을 정의
        2. 허브, 라우터, 네트워크 카드, 케이블 등을 통해 **비트**를 전송
        3. 상위 계층에서 전송된 물리 매체를 통해 다른 시스템에 전기적 신호로 전송
        4. (리피터, 허브)
    2. <데이터 링크 계층>
        1. 물리적 링크를 이용해 신뢰성 있는 데이터를 전송, 네트워크를 통해 데이터를 전송할 때 전송로 역할
        2. 비트를 프레임이라는 논리적 단위로 구성
        3. 전송하려는 데이터에 인접하는 노드(시스템)의 주소(LAN에서 MAC주소를 씀)가 더해짐 - (주소는 최종 수신지의 주소가 아닌 전송되는 다음 노드의 주소)
    3. <네트워크 계층>
        1. 상위 계층에 연결하는데 필요한 데이터 전송과 경로 선택 기능을 제공하고, 라우팅 프로토콜을 사용해 최적의 경로를 선택
        2. 데이터를 전송할 수신 측의 주소를 찾고 수신된 데이터의 주소를 확인해 내 것이면 전송 계층으로 전송(이 때 주소는 IP)
        3. 데이터를 패킷 단위로 송신 측에서 분할해 전송한 후 수신 측에서 재결합
        4. 분할하는 사이즈는 MTU
        5. **(중요)3 계층에 헤더와 데이터에 패킷에 최대 크기가 1500바이트**
    4. <전송 계층>
        1. 여기부터는 물리를 벗어나 어플리케이션(프로그램)에 가까워짐
        2. 프로토콜(TCP, UDP)과 관련된 계층으로 오류 복구와 흐름 제어 등을 담당하며, 두 시스템 간에 신뢰성 있는 데이터를 전송
        3. 네트워크 계층(3 계층)에서 온 데이터를 세션 계층(5 계층)의 어느 어플리케이션에 보낼 지 판독하고, 네트워크 계층으로 전송할 경로를 선택
        4.  네트워크 계층(3 계층)에서 전송한 데이터와 실제 운영체제의 프로그램이 연결되는 통신 경로
    5. <세션 계층>
        1.  응용 프로그램 계층 간의 통신을 제어하는 구조를 제공하려 응용 프로그램 계층 사이의 접속을 설정,유지,종료 시키는 역할
        2. 사용자와 전송 계층 간의 인터페이스 역할, LAN 사용자가 서버에 접속할 때 이를 관리하는 기능
        3. 통신 장치 간의 설정을 유지하고 동기화 하는 역할
    6. <표현 계층>
        1. 데이터 표현 차이를 해결하려고 서로 다른 방식으로 변환하거나 공통 형식을 제공
        2. 송신 측에서는 수신 측에 맞는 형태로 변환, 수신 측에서는 응용 계층에 맞는 형태로 변환
        3. 송신 측과 수신 측 사이에서 표준화 된 데이터 형식을 규정
    7. <응용 계층>
        1. 파일 전송, 데이터베이스, 원격 접속, 이메일 전송 등 응용 서비스를 네트워크에 접속 시키는 역할을 함
        2. 사용자에게 정보를 입력 받아 하위 계층으로 전달하고, 하위 계층에서 전송한 데이터를 사용자에게 전달
        3. 실제로 통신의 최종 목적에 해당하는 가장 중요한 계층
        4. 응용 계층에서는 헤더와 트레일러가 추가되지 않음
3. **각 계층 별 기능**
    1. 물리 계층 : 물리적으로 데이터를 전송하는 역할을 수행
    2. 데이터 링크 계층 : 물리적 전송 오류를 해결 (오류 감지/ 재전송 기능)
    3. 네트워크 계층 : 올바른 전송 경로를 선택(혼잡 제어 포함)
    4. 전송 계층 : 송수신 프로세스 사이의 연결 기능을 지원
    5. 세션 계층 : 대화 개념을 지원하는 상위의 논리적 연결을 지원
    6. 표현 계층 : 데이터의 표현 방법
        1. 압축 : 전송되는 데이터의 양
        2. 암호화 : 전송되는 데이터의 의미
    7. 응용 계층 : 다양한 응용 환경을 지원
4. 네트워크 장비
    1. **repeater는 1 계층 장비**
        1. 접속 시스템의 수를 증가시키거나 네트워크 전송 거리를 연장하려고 사용
    2. **switch, bridge(안 씀) 2 계층 장비**
        1. bridge : 두 개 이상의 근거리 통신망을 연결해 하나의 네트워크로 만들어 주는 장치
        2. 3계층용,4계층용,7계층용 등에 switch존재
    3. **router 3 계층 장비**
        1.  LAN과 LAN을 연결
        2. 컴퓨터도 3 계층으로 생각
    4. 유선 전송 매체
        1. ex) 100BASE-T
            - 100 → 속도 (단위: Mbps)
            - BASE → baseband (디지털 방식), BROAD → Broadband (아날로그 방식)
            - T → Twisted pair cable(케이블 종류) / 숫자일 경우 최대 통신 거리
        2. 꼬임선 : 이중 나선 케이블
            1. 스트레이트 케이블 : 양쪽 선에 배열이 같음
            2. 크로스 오버 케이블 : 한쪽은 T568A 배열, 다른 쪽은 T568B 배열
            3. STP(Spanning Tree Protocol)
                
                ![image.png](attachment:8ec21da7-6391-4f80-8a0d-c3e6f660faa8:image.png)
                
            4. Auto-MDI-X
                
                ![스크린샷 2026-01-28 205449.png](attachment:532f4417-86db-4f77-b04b-ad7938b6b4df:스크린샷_2026-01-28_205449.png)
                
        3. 동축 케이블
            1. 꼬임선보다 주파수가 높고 데이터 전송이 빠르다.
            2. 외부 신호와 전자파를 차단하는 능력이 뛰어남.
            3. 케이블 가운데 있는 와이어와 와이어를 감싸는 차폐용 그물망 이렇게 두 개의 전도체로 구성됨.
        4. 광섬유 케이블
            1. 빛을 이용해 데이터를 전송하기 때문에 휘는 범위에 한계가 있다.
            2. 설치가 어렵고 가격이 비싸다.
            3. 매우 높은 대역 용량을 제공하고, 신호 감쇠에 영향이 없다.
            4. EMI 영향을 받지 않고, 내구성이 좋다.
