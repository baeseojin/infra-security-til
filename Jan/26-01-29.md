## 📝 오늘 배운 내용 요약

1. 유니캐스트
    - 네트워크에서 가장 많이 사용하며 서버와 클라이언트 간의 일대일 통신 방식
    - 통신하려면 전송되는 프레임 안에 항상 송신지와 수신지 주소, 주 MAC 주소가 있어야 한다.
    - 자신과 수신지의 MAC 주소가 동일하면 전송된 데이터를 수신, 아니면 해당 프레임은 버린다.
    - **가장 많이 사용하는 통신 방식으로 수신지 주소(MAC 주소)를 적어 특정 컴퓨터에 전송**
2. 브로드캐스트
    - 로컬 LAN(라우터로 구분된 공간)에 있는 모든 네트워크 단말기에 데이터를 보내는 방식
    - 서버와 클라이언트 간에 1:모두로 통신
    - 주소는 FF-FF-FF-FF-FF-FF로 미리 정해져 있음.
    - 다른 라우터를 찾거나, 라우터끼리 데이터를 교환하거나, 서버가 서비스를 제공하려고 모든 클라이언트에게 알릴 때 등에 사용
    - 수신을 원치 않는 클라이언트도 수신하게 되므로 네트워크 성능 저하를 가져올 수 있음.
    - **영역 안에 있는 모든 컴퓨터에 한 번에 다 전송**
3. 멀티캐스트
    - 데이터를 무조건 CPU로 전송해 컴퓨터 자체의 성능을 떨어뜨리는 브로드캐스트에 문제를 쉽게 해결
    - 전송하려는 특정 그룹에게만 한 번에 전송할 수 있기에 **유니캐스트처럼 반복해서 보내지도**, **브로드캐스트처럼 전송 받을 필요가 없는 컴퓨터에 보내지**도 않음
    - **특정 그룹 컴퓨터에만 한 번에 데이터를 전송해 그룹 이외의 컴퓨터에 영향을 주지 않음**
4. 애니캐스트
    - 논리적인 거리가 가깝다(네트워크 속도와 관련)
    - 거리가 가까운 시스템 1개와 통신(유니캐스트와 유사)
    - DNS(Domain Name System), Cdn(Content delivery networks)에 사용

1. 정보 흐름의 방향성
    1. 단방향(Simplex)
        1. 송신측과 수신 측이 미리 고정되어 있고, 통신 채널을 통해 접속된 단말기 두 대 사이에서 데이터가 한쪽 방향으로만 전송되는 통신 방식
    2. 양방향(Duplex)
        1. 통신 채널을 통해 접속된 두 대의 단말기 사이에서 데이터의 송수신이 모두 가능한 방식
            - 전이중(Full-Duplex)
                - 통신 채널에 접속된 단말기 두 대가 동시에 데이터를 송수신 할 수 있는 통신 방식
                - **요즘은 거의 전이중 사용**
            - 반이중(Half-Duplex)
                - 통신 채널에 접속된 두 대의 단말기 중 어느 한쪽이 데이터를 송신하면 상대방은 수신만 할 수 있는 통신 방식
    3. 통신방식
        - 직렬 전송
            - **이제 대부분 직렬 전송 씀**
        - 병렬 전송
2. MAC 주소
    1. 전송 매체에 접근 제어할 수 있는 주소
    2. NIC칩에 주소가 박혀있다.
    3. octets = 8비트 = 1바이트
    4. 6octets
        1. 3 octets(OUI) 
        2. 3 octets(NIC)
