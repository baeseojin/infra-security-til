## 📝 오늘 배운 내용 요약

1. SELinux
    - 프로세스와 리소스 간에 허용된 작업을 명시적으로 정의하는 액세스 규칙 집합을 적용, 액세스 규칙에 정의되어 있지 않은 작업은 허용되지 않음
    - 작동 모드
        - Enforcing(적용) : SELinux 정책을 적용하며, 위반 시 접근을 차단하는 모드 (기본값)
        - Permissive(허용) : SELinux 정책을 적용하되, 접근을 차단하지 않고 로그만 남김
        - Disable(비활성화) : SELinux 비활성화, 정책 적용과 로그 기록이 모두 이루어지지 않음
        - `getenforce` : selinux 설정 상태 확인
        - `setenforce  0 or 1` → enforcing 또는 permissive : selinux 설정 변경
        - `vim /etc/selinux/config` : 영구 설정(SELINUX부분을 enforcing 또는 permissive로 수정)
    - context 확인
        - file/directory
            - `ls -Z file`
            - `ls -Zd dirctory`
        - process
            - `ps -eZ | grep 패턴`
        - user
            - `id -Z 사용자`
    - context 변경
        - `chcon -t 유형 file/dirctory`
            - 유형 작성 시 오타가 나는 경우가 있음
            
            ```powershell
            ex)
            
            touch file1
            mv file1 /var/www/html/
            ls -lZ /var/www/html/file1
            ls -dlZ /var/www/html/
            chcon -t httpd_sys_content_t /var/www/html/file1
            ls -lZ /var/www/html/file1
            ```
            
        - `restorecon -v file/directory`
            - 컨텍스트를 지정하지 않고, SELinux 정책에 맞는 규칙을 사용하여 컨텍스트 수정
            
            ```powershell
            ex)
            
            touch file2
            mv file1 /var/www/html/
            ls -lZ /var/www/html/file2
            restorecon -v /var/www/html/file2
            ls -lZ /var/www/html/file2
            ```
            
    - 새 파일은 해당 파일 이름이 기존 레이블 지정 정책과 일치하는 경우 기본 레이블을 가져옴
    - 새 파일의 이름이 기존 레이블 지정 정책과 일치하지 않는 경우에는 파일에 상위 디렉터리와 동일한 레이블이 상속
    - 복사 같은 경우는 해당 경로에 파일을 새로 만들기 때문에 경로의 상위 디렉터리 SELinux context를 상속받음
    - 이동 같은 경우는 파일을 옮기기 때문에 SELinux context가 그대로임
    - SELinux 기본 파일 컨텍스트 정책 정의
        - `semanage fcontext`
            - 기본 파일 컨텍스트를 결정하는 정책을 표시하고 수정
            - `semanage fcontext { -a | -d | -l } -t [context] [path]`
                - `-a` : --add 지정한 개체 유형의 기록 추가
                - `-d` : --delete 지정한 개체 유형의 기록 삭제
                - `-l` : --list 지정한 개체 유형의 기록 나열
            
            ```powershell
            ex)
            
            mkdir /virtual 
            touch /virtual/index.html 
            ls -dlZ /virtual
            ls -lZ /virtual/
            semanage fcontext -a -t httpd_sys_content_t '/virtual(/.*)?’ 
            restorecon -RFvv /virtual/
            ls -dlZ /virtual/
            ls -lZ /virtual/
            ```
            
    - selinux bool
        - 해당 selinux 부울 확인
            - `getsebool (정책명) -a`
        - selinux 부울 변경
            - `semanage (정책명) -l`
                - 부울의 런타임 설정과 영구 설정을 보여줌
            - `setsebool [정책명] [on | off]`
                - runtime 변경
            - `setsebool  -P [정책명] [on | off]`
                - permanent 변경
                - `-P` : 영구적으로 설정
        
        ```powershell
        ex)
        
        semanage boolean -l | grep httpd_enable_homedirs
        setsebool httpd_enable_homedirs on
        semanage boolean -l | grep httpd_enable_homedirs
        getsebool httpd_enable_homedirs
        semanage boolean -l | grep httpd_enable_homedirs
        setsebool -P httpd_enable_homedirs on
        semanage boolean -l | grep httpd_enable_homedirs
        ```
        
    - selinux port label
        - 포트레이블 확인
            - `semanage port -l | grep pattern`
        - 포트레이블 설정
            - `semanage port { -a | -m | -d } -t port-type -p { tcp | udp } port-number`
            - -a : 포트 번호에 포트 레이블 지정
            - m : 포트 번호에 포트 레이블 변경
            - -d : 포트 번호에 포트 레이블 제거
            - -t : 포트 레이블 유형
            - -p : 프로토콜 유형 (TCP or UDP)
        
        ```powershell
        ex)
        
        semanage port -l | grep http_port_t
        semanage port -a -t http_port_t -p tcp 8900
        semanage port -l | grep http_port_t
        semanage port -l | grep http_port_t
        semanage port -a -t ssh_port_t -p tcp 8900
        semanage port -m -t ssh_port_t -p tcp 8900
        semanage port -l | grep http_port_t
        semanage port -l | grep ssh_port_t
        semanage port -l | grep ssh_port_t
        semanage port -d -t ssh_port_t -p tcp 8900
        semanage port -l | grep ssh_port_t
        ```
        
2. DB
    - `mariadb-server` : 패키지 설치
    - `mysql_secure_installation` :  mysql 설치
    - `mysql -u (계정명) -p` : mysql 접속
    - DDL (Data Definition Language, 데이터 정의어)
        - DB 구조(테이블, 뷰, 인덱스 등)를 정의, 변경, 삭제
        - CREATE (생성)
            - `CREAT TABLE (테이블명) (컬럼이름 데이터타입 제약조건)`
            - 제약조건
                - NOT NULL
                - UNIQUE
                - PRIMARY KEY
                - FOREIGN KEY
                - CHECK
            - `CREATE USER (사용자이름@호스트이름) IDENTIFIED BY (패스워드)` : 사용자 추가
        - ALTER (수정)
            - `ALTER TABLE (테이블명) ADD (컬럼이름 데이터타입)`
            - `ALTER TABLE (테이블명) DROP (컬럼이름 데이터타입)`
            - `ALTER TABLE (테이블명) MODIFY (컬럼이름 데이터타입)`
        - DROP (삭제)
            - `DROP TABLE (테이블명)`
            - `DROP USER (사용자이름@호스트이름)`
        - TRUNCATE (데이터 초기화)
    - DML (Data Manipulation Language, 데이터 조작어)
        - 테이블에 데이터를 조회, 입력, 수정, 삭제
        - SELECT (조회)
            - `SELECT 컬럼명 FROM 테이블명`
        - INSERT (삽입)
            - `INSERT INTO 테이블명(컬럼명) VALUES (값)`
        - UPDATE (수정)
            - `UPDATE 테이블명 SET 컬럼명 = 값`
        - DELETE (삭제)
            - `DELETE FROM (테이블명)`
    - DCL (Data Control Language, 데이터 제어어)
        - 데이터베이스 접근 권한 및 보안 설정
        - GRANT (권한 부여)
            - `SHOW GRANTS FOR (사용자이름@호스트이름)` : 해당 사용자의 권한 확인
            - `GRANT (명령어) ON (데이터베이스 및 테이블명) TO (사용자이름@호스트이름)`
        - REVOKE (권한 회수)
    - TCL (Transaction Control Language, 트랜잭션 제어어)
        - DML 작업의 논리적 작업 단위(트랜잭션)를 제어
        - COMMIT (영구 반영), ROLLBACK (취소/되돌리기), SAVEPOINT (저장점 설정)
    - `SET autocommit = 0` : 자동 커밋 비활성화
    - 자동 커밋 비활성화 영구 설정
        - `/etc/my.cnf.d/` 안에 cnf 파일 설정
