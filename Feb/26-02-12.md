## 📝 오늘 배운 내용 요약

1. rpm
    - 패키지를 쉽게 설치하기 위한 도구
    - {이름}-{버전}-{릴리스}.{아키텍처}.rpm
        - 아키텍처
            - i386~686 → 32비트 cpu
            - x86_64 → 64비트 cpu
            - src → 소스 코드 파일
            - noarch → 모든 cpu
    - 종속성(의존성) 문제 해결 x
    - 시스템에 rpm 파일이 존재해야 설치 가능
    - RPM 관련 패키지 정보 확인 명령어
        - `rpm -qa` : 시스템에 설치되어 있는 모든 패키지
        - `rpm -qi` : 패키지에 대한 정보
        - `rpm -qc` : 패키지의 설정파일들 보여줌
        - `rpm -qd` : 패키지의 문서(document)를 보여줌
        - `rpm -ql` : 패키지의 모든 파일 보여줌
        - `rpm -qf` : 파일이나 디렉토리가 어떠한 패키지로 인해서 파생되었는지 알려줌.
    - RPM 관련 패키지 설치 및 업데이트
        - `rpm -Uvh 패키지 파일명`
    - RPM 관련 패키지 삭제
        - `rpm -e 패키지명`
2. yum
    - 현재는 DNF
    - dnf로 yum의 심볼릭 링크가 되어있어서 yum도 사용 가능
    - 종속성(의존성) 문제 해결 o
    - yum(dnf) sub명령어
        - `install 패키지명` : 패키지 설치
        - `remove  패키지명` : 패키지 삭제
        - `update  패키지명` : 패키지 업데이트
        - `info   패키지명` : 패키지 정보확인
        - `list`    : repository 의 패키지 리스트 출력
        - `provies file/dir` : 해당 파일이나 디렉토리가 어떠한 패키지로 인해서 파생되었는지 알려줌
            - `rpm -qf` 와 동일한 작용
        - `repolist all` : 현재 시스템에 등록된 **모든 저장소 상태 확인**
        - `repolist` : 현재 시스템에 지정되어 있는 저장소 목록 확인
        - `makecache` : 현재 시스템에 지정되어 있는 저장소에 대한 cache 생성
        - `search [문자]` : 패키지 이름이나 설명에 특정 문자열이 포함된 항목을 검색
        - `localinstall 패키지파일명` : 패키지 파일을 yum으로 설치
    - yum(dnf) groups sub명령어
        - `groups install 그룹 패키지명` : 그룹 패키지 설치
        - `groups remove  그룹 패키지명` : 그룹 패키지 삭제
        - `groups update  그룹 패키지명` : 그룹 패키지 업데이트
        - `groups info  그룹 패키지명` : 그룹 패키지 정보
    - repository 파일 설정(/etc/yum.repos.d/)
        - 확장자가 반드시 .repo 이여만 함.
        - BaseOS 저장소
            - BaseOS 리포지토리는 운영체제의 기반이 되는 최소한의 기본 기능의 코어(핵심, 필수) 패키지들로 구성
            - 즉, 커널 핵심 패키지 그룹으로 하드웨어, 가상머신, 클라우드 인스턴스, 컨테이너에서 실행 가능하도록 하는 패키지들로 구성
        - AppStream(애플리케이션 스트림) 저장소
            - AppStream은 BaseOS의 패키지 외에 어플리케이션, 소프트웨어, 종속성, 데이터베이스과 관련된 패키지들로 구성
            - 즉, Apache웹서버, Cyrus-imapd, Vim 등과 같은 패키지들, 또는 Java, PHP, Python 등과 같은 개발 언어 패키지들, 그리고 MariaDB, PostgreSQL 등과 같은 DBMS 패키지들로 구성된다.
            - AppStream 패키지들은 BaseOS 패키지들보다 훨씬 더 많이 업데이트 됨.
        - 저장소 내용 구성
            
            ```powershell

            baseurl=http://주소
                    file://경로

            gpgcheck=1/0 (1이면 라이센스 키가 있을 경우, 0이면 라이센크 키가 없는 경우)
            gpgkey=file//경로(gpgcheck=1 일 경우에만 작성)
            ```

3. rpm vs yum(dnf)차이점
    - 패키지 파일 유무
        - rpm : 해당 패키지 파일을 가지고 있어야 함
        - yum :  인터넷을 통해 저장소(repository)의 패키지 파일을 다운 받음
    - 종속성(의존성)해결
        - rpm : 해당 패키지를 설치하기 위한 종속성 패키지를 해결 할 수 없음
        - yum : 해당 패키지에 대한 종속성 패키지까지 같이 설치 됨
    - 인터넷 연결
        - rpm : 인터넷 없이 설치 가능 (파일만 있으면 됨)
        - yum : 기본적으로 저장소 사용 → **인터넷 필요**

systemctl list-units -a

systemctl list-unit-files

systemctl status httpd

systemctl status autofs

httpd 서비스를 시작하고 활성화 시키시오.

systemctl enable httpd

or
systemctl enable httpd  --now

systemctl status  httpd

systemctl disable autofs

systemctl status autofs

systemctl stop httpd

systemctl disable httpd
or
systemctl disable httpd --now

systemctl status httpd

systemctl enable autofs

systemctl status autofs

systemctl mask httpd

systemctl status httpd

systemctl start httpd

에러발생 : httpd mask 상태

systemctl unmask httpd

systemctl status httpd

autofs 서비스의 상태를 확인하고 리로드와 재시작하면서 pid 를 확인해보시오.

systemctl status autofs

systemctl start autofs

systemctl restart autofs

systemctl status autofs (pid 확인)

systemctl reload autofs

systemctl status autofs (pid 확인)

systemctl list-dependencies autofs
