## 📝 오늘 배운 내용 요약

1. 파이프
    - 한 프로그램의 출력을 중간 파일 없이 다른 파일의 입력으로 바로 보내는 유닉스 매커니즘
    - `|`-> 파이프
    - `|` 왼쪽 명령어의 출력을 오른쪽 명령어의 입력으로 보냄
    - 파이프 라인은 하나 이상의 파이프로 구성
2. 초기화 파일 = 환경 설정 파일 = 프로파일
    - /etc/profile
        - root만 수정할 수 있는 파일
        - 시스템 전역 환경 설정 파일
        - 모든 계정이 **로그인**할 때 환경 설정을 적용하는 파일
    - ~./profile
        - 개별 계정의 홈 디렉터리에 있는 파일
        - 해당 계정이 로그인할 때 환경 설정을 적용하는 파일
    - /etc/bashrc
        - root만 수정할 수 있는 파일
        - 시스템 전역 환경 설정 파일
        - 모든 계정에서 bash 쉘을 실행할 때 환경 설정을 적용하는 파일
    - ~/.bashrc
        - 개별 계정의 홈 디렉터리에 있는 파일
    - ~/.bash_profile
        - 해당 계정이 bash를 실행할 때 환경 설정을 적용하는 파일
3. 변수
    - 데이터를 저장하고 참고하기 위해 이름을 붙인 메모리 공간
    - `password=1234`로 변수 선언
    - 쉘변수 : 현재 쉘 안에서만 사용 가능한 변수
    - 환경변수 : 현재 프로세스와 그 자식 프로세스들이 참조할 수 있는 변수
    - 특수 변수 : 쉘에서 사용하는 쉘을 돕는 특이 변수
    - 위치 매개 변수 : 인수의 위치 관련 변수
    
    ```powershell
    PATH : 쉘이 명령어를 찾는 디렉터리 경로
    
    HOME : 로그인 시 사용자의 홈디렉터리
    
    PS[번호] : 쉘프롬프트
    
    TERM : 터미널이름
    
    SHELL : 쉘이름
    
    LOGNAME : 로그인 사용자 계정
    
    echo $변수이름 : 변수 값 출력
    
    env : 환경변수 만 출력
    
    set : 환경변수, 쉘변수 출력
    
    export : 쉘 변수를 환경변수로 승격
    
    pass=1 쉘변수
    
    export pass : pass 쉘변수 -> 환경변수 pass로 변경
    
    export pass=1 환경변수로 바로 만들어짐
    
    unset 변수이름 : 변수 삭제
    
    read 변수이름 -> 값
    
    *환경 설정 파일에 저장한 값은 로그인을 새로 하거나 쉘을 새로 실행해야 적용이 됩니다.*
    
    source [환경설정파일] : 재로그인 하지 않고도 환경설정파일 바로 적용
    
    ```
    
4. 프로세스
    - 부모 프로세스 : 다른 프로세스를 생성할 수 있는 프로세스
    - 자식 프로세스 : 부모 프로세스로부터 만들어지는 프로세스
    - 데몬 프로세스 : 우리가 직접 실행할 수도 있지만 (데몬) 리눅스가 보통 실행함.
        - 일반적으로 보통 백그라운드에서 실행됨
        - 특정 서비스를 제공하기 위해 구동
        - 프로세스 이름이 보통 d로 끝남
    - 고아 프로세스 : 자식 프로세스는 종료하면 부모 프로세스로 되돌아감
        - 종료 하기 전에 부모 프로세스가 먼저 종료된 상태.
    - 좀비 프로세스 : 정상적으로 종료가 되지 않은 프로세스
        - 이미 종료가 되었지만 리소스 반환을 안하고 남아있는 프로세스
    - `ps` : 현재 실행 중인 프로세스 정보 출력
    - `ps -ef` :  프로세스 상세 정보 출력
        - `-e` 모든 프로세스 출력
        - `-f` 상세정보 출력
    - `ps aux`
        - USER : 프로세스를 실행한 사용자의 이름
        - PID : 프로세스 ID
        - %CPU : 프로세스의 현재 CPU 사용률
        - %MEM : 프로세스의 현재 메모리 사용률
        - VSZ : 프로세스가 사용중인 가상 메모리의 크기
        - RSS : 프로세스가 사용중인 실제 메모리의 크기
        - TTY : 프로세스가 연결한 터미널 장치파일 이름
        - STAT : 프로세스의 상태 코드
            - R : 실행중
            - S : 수면상태
            - T : 정지 상태
            - Z : 좀비 상태
        - START : 프로세스가 시작된 시간
        - TIME : 프로세스를 CPU를 사용한 총 시간
        - COMMAND : 프로세스가 실행한 명령어
    - `pgrep [프로세스이름]` :  pid 출력
    - `pgrep -l` : pid도 출력, 프로세스 이름도 출력
    - `pgrep -u uid` : 해당 uid계정이 실행중인 프로세스 pid
    - `pgrep -t pst/0` : 해당 터미널에서 실행중인 프로세스
    - `pgrep -n [프로세스이름]` : 가장 최근에 생성한 프로세스 출력
    - `pstree` :  프로세스 실행 정보를 트리 형태로 출력, d로 끝나면 데몬 프로세스 실행
5. 시그널
    - 프로세스에 보내는 간단한 신호
    - `kill -숫자 프로세스pid`
        - 예) `kill -9 2086` : 2086 pid를 가진 프로세스에게 9번 시그널을 보내라
    - `pkill -번호 [프로세스명]`
    - 주요 시그널 번호
        - 1 - `SIGUP` : Hang up 터미널이 연결 종료되었음을 프로세스에 알림
        - 2 - `SIGINT` : Interrupt = ctrl +c
        - 9 - `SIGKILL` : Kill 강력하게 종료 신호
        - 15 - `SIGTERM` : Terminate 일반적인 종료 신호
    - 번호 생략 시 15번 시그널을 보냄
    - 특정 프로세스는 15번을 무시하는 경우가 있고 9를 보내면 거의 종료된다.
6. 작업
    - 프로세스중에서 쉘이 관리하는 프로세스
    - 3가지상태로 작업을 관리함
        - Foreground Job
        - Background Job
        - Stopped jobs
    - `jobs` : 현재 작업 중인 목록 출력
    - `fg %[작업번호]` :  해당 작업을 Foreground로 실행
    - `bg %[작업번호]` :  해당 작업을 Background로 실행
7. 아카이브
    - `tar`
        - v : 화면에 진행과정을 자세하게 출력
        - f : 아카이브 파일을 지정하는 옵션
        - c : 아카이브 하기
        - t : 아카이브 내용 보기
        - x : 아카이브 풀기
        - z : 압축(원래 압축 기능이 없었지만 최신 리눅스에서 추가됨)
    - 예) `tar -cvf [아카이브명] fiel1 file2 file3`
    - `tar zxOf number.tar.gz`: 안에 파일 내용을 읽을 수 있음
    - `tar zcvf text.tar.gz file1 file2 file2` : 아카이브 및 압축을 동시에 하기
    - `tar zxvf text.tar.gz` : 아카이브 및 압축을 동시에 해제하기
8. 압축
    - 압축 명령어 : `gzip`, `bzip2`, `zip`
    - `gzip`
        - `gzip number` : 압축하기
        - `zcat number.gz` : 압축한 파일 내용 보기
        - `gunzip number.gz` : 압축풀기
    - `bzip2`
        - `bzip2 1` : 압축하기
        - `bzcat 1.bz2` : 압축한 파일 내용 보기
        - `bunzip2 1.bz2` : 압축풀기
    - `zip`
        - `zip number.zip 1 2 3` : 압축하기
        - `unzip number.zip` : 압축 풀기
    - 성능이 gzip이 zip보다 좋고 많이 쓴다.
