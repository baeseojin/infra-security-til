## 📝 오늘 배운 내용 요약

1. 확장 권한
    - 실행 권한이 반드시 있어야 한다
    - setuid
        - 파일 o
        - 디렉터리 x
        - 파일의 소유주 권한으로 실행
        - 심볼릭 모드 : `chmod u+s file명`
        - 8진수 모드 : `chmod +4000 file명`
    - setgid
        - 파일 o
            - 파일의 소유그룹 권한으로 실행
        - 디렉터리 o
            - setgid 부연된 디렉토리의 파일들에 대해서 소유 그룹이 계속해서 소유 그룹 권한을 행사함.
            - setgid는 하위 새로 만들어지는 디렉토리에 상속
        - 심볼릭 모드 : `chmod g+s file/dir명`
        - 8진수 모드 : `chmod +2000 file/dir명`
    - sticky bit
        - 파일 x
        - 디렉터리 o
        - 해당 디렉토리에 생성된 파일이나 디렉토리에 대한 삭제 권한을 파일의 소유자와 root에게만 가능하도록 함.
        - 심볼릭 모드 : `chmod o+t dir명`
        - 8진수 모드 : `chmod +1000 dir명`
    - 숫자표
    
    | 숫자 | 의미 | 이름 |
    | --- | --- | --- |
    | 1 | Sticky bit | 스티키 |
    | 2 | SetGID | SGID |
    | 4 | SetUID | SUID |
    | 0 | 없음 | - |
    - 실습
    
    ```powershell
    <확장권한 실습>
    
    ptest 디렉토리 생성
    
    mkdir /ptest
    
    /ptest에 모든 권한을 rwx로 주고 setgid 권한도 줄 것
    
    chmod a=rwx,g+s /ptest
    
    or
    
    chmod 2777 /ptest
    
    user01로 사용자 전환
    
    su - user01
    
    /ptest 안에 파일 file01 과 디렉토리 dir01 생성 
    
    cd /ptest
    touch file01
    mkdir dir01 
    
    file01과 dir01 소유권 확인 및 dir01의 퍼미션 확인
    
    ls -ld file01 dir01
    
    => 소유그룹이 root 으로 되어 있고 dir01에도 setgid 설정되어 있음.
    
    root로 사용자 전환
    
    exit
    
    dir01의 소유권을 user01로 바꾸고 소유그룹을 student로 변경 하고 sticky bit 권한과 모든 사용자에게 rwx 권한을 줄것 
    
    cd /ptest/
    groupadd student
    chown user01:student dir01     -> chown은 소유자 변경 명령
    chmod 1777 or a=rwx,o+t dir01
    
    file01에 setuid 권한을 줄 것
    
    chmod u+s file01
    
    user02로 사용자 전환
    
    su - user02
    
    ./dir01 안에 파일 file02와 디렉토리 dir02 생성
    
    cd /ptest/dir01
    mkdir dir02
    touch file02
    
    file02 와 dir02의 소유권 확인
    
    file02 소유자 : user02 소유그룹 : student
    dir02  소유자 : user02 소유그룹 : student
    
    user03로 사용자 전환 /ptest/dir01/file02 삭제 해볼 것
    
    su - user03
    
    /ptest/file01 삭제 해볼 것
    
    rm -rf /ptest/file01
    삭제됨
    
    rm -rf /ptest/dir01/file02
    삭제 안됨 : sticky bit 으로 인해서 삭제 불가
    
    setgid 권한이 주어진 파일이나 디렉토리를 검색하는 방법은?
    
    find / -perm -2000 2> /dev/null
    
    or
    
    ls -lR / | grep '^......s.... '
    ```
    
2. 작업 예약
    - 단일성 예약
        - 특정시기에 한번 작업을 수행
        - atd 서비스를 이용
        - `at timespec`
            - at> 프롬프트가 나오면 해당 작업 작성 후 엔터
            - at> 프롬프트에 명령어 입력 후 `ctrl + d` 를 누르면 예약 완료
            - `at -l or atq` : 예약 확인
            - `at -c 예약번호` : 작업 내용 확인
            - `atrm 예약번호` : 작업예약 취소
        - timespec 작성 요령
        
        ```powershell
        at 시간 날짜 순으로 기입
        
        22년 2월 23일 오전 10시 기준
        
        지금으로부터 2시간 뒤 작업
        
        at now +2hour (분 - min, 시 - hour)
        
        오후 4시 작업
        
        at 16:00 
        at 04:00 PM
        at teatime
        at 16:00 2/23/22 (월,일,년)
        at 16:00 23-02-23
        at 04:00 PM today
        at teatime today
        
        새벽 12시 작업
        
        at 00:00 
        at 12:00 AM
        at midnight
        
        내일 정오 작업
        
        at 12:00 PM 2/24/22
        at noon tomorrow
        	
        /usr/share/doc/at/timespec 참조
        ```
        
        - at 실습
        
        ```powershell
        <at 실습>
        
        1.3분 뒤에 ps -ef 명령어를 실행해서 psfile01에 저장되도록 설정
        
        at now +3min
        at> ps -ef > psfile01
        at> crtl+d
        
        2.24년 10월 1일에 date 명령어를 실행해서 메일로 오도록 설정
        
        at 00:00 10/1/24
        at> date
        at> crtl+d
        
        3.오늘 오후 12시에 cal 명령어를 실행해서 메일로 오도록 설정
        
        at 12:00 PM today
        at> cal
        at> crtl+d
        
        4.내일 오후 4시에 ps -ef 명령어를 실행해서 psfile02에 저정하도록 설정
        
        at teatime tomorrow
        
        5.작업 예약을 확인하시오.
        
        at -l
        or
        atq
        ```
        
    - 반복성 예약
        - crond 서비스 이용
        - 사용자 cron
            - `crontab` 이라는 명령어를 사용
                - `crontab -e` : 작업예약
                - `crontab -l` : 작업예약 목록 확인
                - `crontab -r` : 작업예약 취소(모든 작업 취소)
                - `crontab 파일명` : 다른 파일에 만들어놓은 작업을 불러올 수 있음.
            - cron의 timespec 설정법
            
            ```powershell
            X      X      X       X         X         command
            분     시     일      월        요일
            (0~59) (0-23) (1-31)  (1-12)    (0-7:0,7 일요일)
                                  (약식월)  (약식요일)
            특수문자
            * : 모두 참
            - : 범위
            / : 주기
            , : 연속되지 않은 다수
            
            1월1일 새벽12시 reboot
            
            0 0 1 1 * reboot
            
            12월 15일 오후 3시부터 오후 6시까지 15분마다 date
            
            */15 15-18 15 12(or Dec) * date
            
            매월 첫번째 월요일 오후 1시 정각에 date 
            
            0 13 1-7 * 1(Mon) date
            
            매분마다 date명령어를 datefile에 기입하시오.(이어쓰기로)
            
            * * * * * date >> datefile
            
            1분마다
            */1
            ```
            
        - 시스템 cron
            - cron
                - 일반 cron은 시스템이 켜져 있는 동안에만 해당 시간의 작업을 수행
                - 해당 시간에 시스템이 다운되어 있으면 수행하지 않음.
                - /etc/crontab 파일을 양식지로 사용해서 /etc/cron.d/ 다른 파일 저장으로 등록
                - 기존 사용자 cron과 다른 점 cron 예약시 사용자 계정이 들어감.
            - anacron
                - anacron은 작업 해당 시간에 시스템이 꺼져 있어서 실행하지 못하더라도 시스템이 켜지면 일정 시간 대기 후에 해당 작업 실행
        - 실습
        
        ```powershell
        1.매년 1월 첫번째 일요일 자정에 재부팅이 되도록 설정
        
        0 0 1-7 1 0 /sbin/reboot
        
        2.매월 오후 1시부터 5시까지 10분마다 date 명령어를 실행해서 datefile01에 이어쓰기로 저장하도록 설정 
        
        */10 13-17 * * * date >> datefile01
        
        vi 편집기로 test 파일에 다음 내용 설정
        
        vim test
        
        3.매년 3월, 6월, 9월 2번째 화요일 14시 20분에 /etc/passwd 화일의 내용을 userfile에 갱신하도록 설정
        
        20 14 8-14 3,6,9 2 cat /etc/passwd > userfile
        
        4.매주 수~금 오후 5시에 /var/log/secure의 내용 중 오전 8시 정각부터 오후 5시59분까지의 내용을 securefile에 갱신하도록 설정
        
        0 17 * * 3-5 egrep '... .. '0[89]|1[0-7]' /var/log/secure > securefile      -> [... ..는 날짜 건너뜀, '0[89]|1[0-7]'는 08,09,10...17시임]
        
        :wq
        
        5.3,4번의 작업을 파일로 예약하시오.
        
        vi test
        
        crontab test
        
        6.작업 예약을 확인하시오.
        
        crontab -l
        
        7. 3번 작업만 삭제하시오.
        
        crontab -e
        3번 작업 삭제
        :wq
        
        8. 모든 작업을 삭제하시오.
        
        crontab -r
        ```

```powershell
﻿초기화로 스냅샷

useradd vagrant

vi /etc/default/useradd

/home => /home1
```
