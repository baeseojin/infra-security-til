## 📝 오늘 배운 내용 요약

1. 네트워크 복습 마지막
    - tcp : 세션을 맺고 통신을 하는 프로토콜
        - tcp 세션을 맺는 방법 : 3way handshake
        - tcp 세션을 끊는 방법 : 4way handshake
    - netstat -an 명령어를 통해  알 수 있음
    - 서버가 포트를 열고 대기 중 -> 클라이언트가 서버에 연결하려고 syn -> 서버가 syn+ack -> 클라이언트가 ack -> 실시간 통신
    - 4handshake는 fin,ack > ack >fin,ack>ack
    - seq : 현재 시퀀스 넘버
    - next  seq : 다음 보낼 시퀀스 넘버
    - tcp가 깨지지 않고 데이터를 보내는 원리
    - tcp는 next seq와 받은 ack 값이 같은지 확인해서 같으면 문제가 없음
    - 호스트이름,서브도메인 = www
    - 도메인 = [google.com](http://google.com/)
    - 프로토콜하고 서비스는 다름
    
    ```powershell
    nslookup
    ```
    
2. 리눅스 기초
    - 터미널: 사용자랑 상호작용하는 인터페이스(로컬,원격)
    - 콘솔: 시스템에 직접연결된 특별한 터미널(로컬)
    - 쉘: 터미널에서 받은 명령어를 해석해 커널에 전달하고 결과를 돌려주는 프로그램
    - 사용자와 OS 사이에서 소통을 담당하고 명령어 실행을 담당
    - 터미널 없이도 백그라운드에서 동작가능
    - 명령어 : 어떤 작업을 하는 실행 파일, 또는 쉘 내장 명령어
    - 옵션: 명령어를 어떻게 실행할지 지정
    - 아규먼트(인수) : 명령어 실행에 영향을 받는 파일, 디렉터리, 텍스트 등 항목

```powershell
ctrl + c : 강제 종료
ctrl + d : 정상 종료
ctrl +l : 화면 제거
ctrl 왼쪽 , ctrl 오른쪽 : 단어기준으로 이동
ctrl + u : 다 지우기

which : 명령어의 경로 확인
date : 날짜보는 명령어
cal : 달력보는 명령어

whoami : 현재 시스템을 사용하고 있는 자신이 누구인가를 확인할 때 사용

users : 시스템에 로그인한 모든 사용자명 출력

쉘 내장 명령어 예시 : cd ,echo, pwd, export, alias, source

uname -s 커널의 이름, 운영체제 종류
uname -r 커널의 버전, 운영체제 릴리즈
uname -a 커널의 모든 정보
uname -v 운영체제 버전 출력
uname -n 호스트 이름

whatis 명령어 : 지정한 키워드나 명령어의 메뉴얼 페이지를 찾아서 간략하게 보여줌.

명령어 --help

명령어 -h 도움말빨리 확인하기

man : 메뉴얼도움말을 여는 명령어

mandb -> man 명령어 업데이트 (안의 내용물 최신화)

man -k 문자열

man -f 문자열 정확히 일치하는 것 검색

man 번호 문자열 : 특정 섹션번호 도움말 열기

pwd : 현재 위치하는 경로 확인

cd : 원하는 디렉토리로 이동

clear : 화면상에 있는 내용을 청소할 때 사용

중요한 명령어 : ls -al
```
