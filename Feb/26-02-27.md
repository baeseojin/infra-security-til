## 📝 오늘 배운 내용 요약

1. NFS(Network File System)
    - 분산 파일 시스템 프로토콜
    - 윈도우의 공유 폴더처럼 파일을 공유하여 사용할 수 있음(파일의 일관성 유지)
    - 여러 시스템이 동일 데이터를 사용하여 **데이터 일관성 유지 가능**
    - 파일을 공유할 때 접근할 수 있는 호스트 목록 지정 가능
    - 커버로스와 함께 사용해 인증, 무결성 검증, 암호화 통신 사용 가능
2. NFSv4
    - rpcbind를 사용하지 않고 2049/TCP 포트 대기
    - 마운팅과 잠금 프로토콜 통합 → 방화벽 구성 쉬워짐
    - 성능 향상 & 보안 강화
    - pNFS 사용 가능
    - 하위 버전과 호환
3. NFS 서버 구성
    - `nfs-utils` 패키지 설치
        - `dnf -y install nfs-utils`
    - 공유 디렉토리 생성 및 권한 설정
        - ex) `mkdir -m 777 -p (경로)`
    - `/etc/exports` 파일에 공유 경로, 접근 제어, 옵션 설정
        
        <img width="496" height="90" alt="image" src="https://github.com/user-attachments/assets/7dd87add-8158-4d4b-8918-996fc492b784" />

        
        - `/etc/exports` 파일 옵션
        
        <img width="928" height="453" alt="image" src="https://github.com/user-attachments/assets/10d17048-1dc0-4ce6-924b-2e4039500e3d" />

        
    - `exportfs -r` 로 적용
        - or 서비스 재시작으로 변경사항 반영
    - `systemctl enable nfs-server --now`
        - NFS 서버 서비스를 즉시 실행하고 부팅 시 자동으로 시작되도록 설정
    - `firewall-cmd --add-service=nfs --permanent`
        - 방화벽 설정
4. NFS 클라이언트 구성
    - `nfs-utils` 패키지 설치
        - `dnf -y install nfs-utils`
    - `df -Th` : 확인 명령어
    - 연결 방식
        - **수동 마운트 (mount)**
            - 영구적이지 않음
            - 테스트 마운트할 때 유용
            - 재부팅하면 해제됨
            - `mount -t nfs -o (옵션) (NFS 서버 주소):(경로) (마운트할 경로)`
                - `-t nfs` 옵션은 NFS 파일 시스템 유형을 지정
                - `-o syns` 옵션은 내보낸 파일 시스템에 대한 모든 트랜잭션이 동기적으로 수행되도록 지정
        - **영구 마운트 (/etc/fstab)**
            - `/etc/fstab` 파일을 편집해 마운트 항목 추가
                - `(NFS 서버 주소):(경로) (마운트할 경로) nfs (옵션[rw,sync]) 0 1`
            - 부팅 시 자동 연결
            - 다수 클라이언트 연결 시 자원 낭비 가능
        - **자동 마운트 (autofs)**
            - `autofs` 패키지 설치
            - 접근 시 자동 마운트
            - 일정 시간 미사용 시 자동 해제
            - 시스템 자원 효율적 사용 가능
            - 맵 파일 형식에 따라 마운트 포인트를 자동으로 생성
            - FS automount(autofs) 구성단계
                
                ```powershell
                1. 패키지 설치
                # dnf install autofs nfs-utils
                
                2. 마스터맵 작성(마운트 방식, 매핑파일 선언)
                # vim /etc/auto.master.d/마스터맵명.autofs
                
                3. 매핑파일 작성
                # vim /etc/auto.매핑파일명
                
                3-1. (직접매핑의 경우에만) 로컬마운트 포인트 생성
                # mkdir PATH
                
                4. 서비스 시작 및 활성화(영구설정)
                # systemctl enable autofs.service --now
                ```
                
            - `/etc/auto.master.d/` 에 맵 파일 구성
                - 마스터맵 파일 작성 요령
                    
                    ```powershell
                    # vim /etc/auto.master.d/마스터맵명.autofs
                    
                                MP(매핑종류)        매핑파일경로
                    [직접매핑]  /-                  /etc/auto.direct
                    [간접매핑]  /shares             /etc/auto.indirect
                    [간접매핑]  /shares             /etc/auto.indirect_multi
                    와일드카드 
                    
                    매핑파일 작성 요령
                                Connect MountPoint    OPTIONS     SOURCE_LOCATION
                    [직접매핑]  /mnt/doc (직접생성)   -rw,sync    server:/shares/docs
                    [간접매핑]  share01               -rw,sync    server:/shares/share01
                                share02               -rw,sync    server:/shares/share02
                    [간접매핑]  *                     -rw,sync    server:/shares/&
                    와일드카드  
                    ```
                    
                - 직접 매핑
                    - `/etc/auto.master.d/` 안에 `autofs` 파일 생성
                        
                        <img width="548" height="62" alt="image" src="https://github.com/user-attachments/assets/e91d5676-ea6f-4159-b921-6111f2ef64e1" />

                        
                    - `auto.(이름)` 파일 생성
                        - `(마운트할 경로) -(옵션) (서버주소):(경로)`
                        - 첫번째 필드는 마운트 포인트를 절대 경로로 지정
                    - 마운트할 폴더 생성 해줘야 함
                - 간접 매핑
                    - `/etc/auto.master.d/` 안에 `autofs` 파일 생성
                        
                        <img width="538" height="67" alt="image" src="https://github.com/user-attachments/assets/be1c4f76-b38a-4e97-8ee7-10d14ca14ebd" />

                        
                        - `(마운트할 상위경로) (파일 이름)`
                        - 첫번째 필드는 마운트 포인트를 지정할 때 상대 경로로 지정
                    - `auto.(이름)` 파일 생성
                        - `(하위 경로) -(옵션) (서버주소):(경로)`
                    - 굳이 폴더 생성하지 않아도 됨
                    - `/indirect_share/sub_indirect` 로 마운트
                - 와일드카드를 사용한 간접 매핑
                    - `/etc/auto.master.d/` 안에 `autofs` 파일 생성

                        - 첫번째 필드는 ‘*’문자가 지정
                        - 세번째 필드는 NFS서버의 주소와 공통된 경로 그리고 ‘&’문자를 지정
                    - `auto.(이름)` 파일 생성

## * 이번 시간에 배운 NFS 서버 구성 실습

```powershell
nfs 서버 구성

nmcli con add con-name nfs-server ifname ens160 type ethernet ip4 192.168.100.150/24 gw4 192.168.100.2 ipv4.dns 8.8.8.8

nmcli con up nfs-server

yum -y install nfs-utils

systemctl enable nfs-server.service --now

firewall-cmd --add-service=nfs --permanent

firewall-cmd --reload

- 공유 디렉토리 생성

mkdir -m 777 -p /shares/{public,direct,indirect}

vi /etc/exports

+++++++++++++++++++++++++++++++++++

/shares/public        192.168.100.0/24(rw,sync)
/shares/direct        192.168.100.0/24(rw,sync)
/shares/indirect      192.168.100.0/24(rw,sync)

+++++++++++++++++++++++++++++++++++

systemctl restart nfs-server.service

exportfs -v
=> 공유 사항 확인

========================================

nfs client 구성

yum -y install nfs-utils

- nfs 서버 확인

mount 192.168.100.150:/  /mnt/

ls /mnt

ls /mnt/shares/

umount /mnt/

df -Th

- mount 명령어를 통한 마운트

mkdir /mnt/client_public

mount -t nfs -o rw,sync 192.168.100.150:/shares/public /mnt/client_public/

df -Th

touch /mnt/client_public/client_file

umount /mnt/client_public

df -Th

- /etc/fstab를 통한 마운트

vi /etc/fstab

++++++++++++++++++++++++++++

192.168.100.150:/shares/public   /mnt/client_public   nfs   rw,sync   0   1

++++++++++++++++++++++++++++

systemctl daemon-reload

mount -a

df -Th

------------------------------------------------------

자동 마운트

yum -y install autofs

- 직접매핑

vi /etc/auto.master.d/direct.autofs

++++++++++++++++++++++++++++++++

/-     auto.direct

++++++++++++++++++++++++++++++++

vi /etc/auto.direct

++++++++++++++++++++++++++++++++

/direct    -rw,sync    192.168.100.150:/shares/direct

++++++++++++++++++++++++++++++++

mkdir /direct

systemctl enable autofs --now

df -Th

ls /direct/

df -Th

touch /direct/client_direct_file

서버 확인 : ls /shares/direct

- 간접 매핑

vi /etc/auto.master.d/indirect.autofs

++++++++++++++++++++++++++++++++

/indirect_share    auto.indirect

+++++++++++++++++++++++++++++++++

vi /etc/auto.indirect

+++++++++++++++++++++++++++++++++

sub_indirect     -rw,sync  192.168.100.150:/shares/indirect

++++++++++++++++++++++++++++++++

ls /

systemctl restart autofs

ls /

ls /indirect_share/

df -Th

ls /indirect_share/sub_indirect

df -Th

touch  /indirect_share/sub_indirect/client_indirect_file

server 확인 : ls /shares/indirect

- 와일드카드를 사용한 간접 매핑

vi /etc/auto.master.d/wild_indirect.autofs

++++++++++++++++++++++++++++++++++

/wild_indirect_share    auto.wild_indirect

+++++++++++++++++++++++++++++++++++

vi /etc/auto.wild_indirect

+++++++++++++++++++++++++++++++++++

*    -rw,sync    192.168.100.150:/shares/&

++++++++++++++++++++++++++++++++++++

systemctl restart autofs
ls /wild_indirect_share/public
ls /wild_indirect_share/direct
ls /wild_indirect_share/indirect
df -Th
```
