## 📝 오늘 배운 내용 요약

1. 소켓과 커널
    - 소켓은 전화기로 비유 가능
    - 운영체제 핵심은 커널
    - 커널이 소켓을 만듬(IP와 포트로 소켓이 만들어짐)
    - 네트워크 통신은 커널이 함
    - 서버에서 어떤 웹 서버가 열리면 웹 서버 프로그램은 IP랑 포트를 가지고 네트워크 접속을 염
    - 커널에 소켓을 생성
        - 리스닝 소켓
    - 클라이언트 서버에 소켓과 서버에 리스닝 소켓이<연결 소켓>을 통해 연결됨
    - 클라이언트마다 연결 소켓으로 각각 연결됨
    - 네트워크 통신에서 서로 통신하는 주체는 프로그램
2. OSI계층 복습

```powershell
7 -> 데이터를 만듬
6 - > 데이터
5 - > 데이터
4 - > 
단위는 세그먼트
3 - > 세그먼트에 헤더를 붙임, 헤더에 ip가 들어있음
단위는 패킷
2 - > 패킷에 헤더가 붙음, 헤더에는 MAC주소가 붙음
단위는 프레임
1 - > 
```

## Wireshark

- ip.addr : 출발지 또는 목적지가 해당하는 것 출력
- ip.src :  출발지만 해당하는 것 출력
- ip.dst :  목적지만 해당하는 것 출력

---

- identifiaction : 식별 번호
- ip flags : 단편화 유무를 알려줌
- Fragment Offset : 단편화된 조각의 위치
- fragment -> 단편화(패킷을 쪼개는것)
- more fragment가 0이면 끝 1이면 뒤에 더 있다

---

- ttl : 패킷의 수명

## TCP와 UDP

- 안정적인 전송이 목표 : tcp
- 연결(세션)을 수립하고 통신 : tcp
- 흐름, 오류체크 : tcp
- tcp - http,telnet, ssh, ftp, pop3, imap 등 대부분의 프로토콜

---

- 단순하고 빠른 전송 : udp
- 연결수립하지 않음 : udp
- 흐름, 오류체크 불가 : udp
- udp - dns, tftp, dhcp,snmp

## ARP 프로토콜

- ARP 프로토콜은 2.5계층
- ARP 프로토콜 : ip주소를 가지고 mac주소를 찾아오는 프로토콜
