## 📝 오늘 배운 내용 요약

1. 사용자 및 그룹 관리 파일
    - /etc/passwd
        - 사용자 정보들을 담고 있다
        - 계정명 : password(X) : UID : GID : 코멘트 : 홈디렉토리 : 로그인쉘
    - /etc/shadow
        - 패스워드를 관리한다
        - 계정명 : 암호화된 패스워드 : lastchd : min : Max : warning : inactive : expire : blank
            - last_changed : 19700년 1월 1일(unix time)부터 몇일이 지났는지에 대한 표기
            - min_date : password를 변경할 수 있는 최소 시간
            - max_date : 현 password를 사용할 수 있는 최대 시간
            - warning_date : 현 password의 max_date에 대한 경고를 만료 몇일 전부터 경고할 것인지
            - inactive_date : password 만료에 대한 유예기간
            - expire_date : 계정에 대한 만료 기간
        - 암호화된 패스워드
            - $해쉬알고리즘 종류(6-sha512) $salt값(패스워드를 변경할때 임의적으로 생성) $ 입력한 패스워드와 salt 값을 해쉬알고리즘으로 암호화한 값
            
            <img width="867" height="447" alt="image" src="https://github.com/user-attachments/assets/b80c2287-e2c4-42b2-b570-b67856de3a67" />

            
    - /etc/group
        - 그룹명 : 패스워드(X) : GID : 보조그룹 구성원
        - 파일이나 디렉토리에 소유그룹의 권한을 행사하기 위해서 사용
        - 기본그룹 : 1인1그룹, 계정생성시에 자동으로 설정
        - 보조그룹 : 0개 이상 가입 가능, 수동으로 설정
    - 사용자 정보 확인

    - 사용자 전환
        - su       : 사용자로만 전환(위치나 일부 환경변수 기존 사용자 정보로 유지)
            - 전환시에 해당 사용자의 password 필요
            - root로 전환시에 해당 password가 알려질 가능성이 있음.
        - su -     : 전환되는 사용자의 모든 환경까지 가져옴.
        - sudo     : 일회성으로 root 권한을 가져올때 사용
            - root password가 아닌 자신의 password 필요
            - sudo를 사용하기 위해서는 wheel 그룹에 속해있어야 함.
            - wheel 그룹이 /etc/sudoers에 sudo를 사용할 수 있는 그룹으로 설정이 되어 있기 때문
        - sudo -i  : su - root 처럼 root로 전환 시 사용
    - 사용자관리
        - 사용자 생성에 따른 설정 파일 2가지
            - /etc/default/useradd
            - etc/login.defs
        - 사용자 생성 기본값 수정 방법 2가지
            - /etc/default/useradd : 해당 내용 확인
            - `vi /etc/default/useradd` 또는 `useradd -D`(default란 의미)
        - 생성
            - `useradd [option] 계정명` => 옵션을 안붙이면 default 설정을 따라감.
            - `-u` : uid 설정(지정하지 않았을 경우 가장 마지막 uid+1)
            - `-g` : gid 설정 (gid를 설정하기 위해서는 반드시 존재하는 그룹이어야지만 설정 가능)
            - `-G` : 보조그룹 설정
            - `-c` : 코멘트 설정
            - `-d` : 홈디렉토리 설정
            - `-s` : 로그인 쉘 설정
                - 서버로의 로그인 필요치 않은 계정(ftp, 메일)일 경우 쉘을 /sbin/nologin 로 설정
            - `-b` : base 홈 변경
            - `-m`(make) : /etc/login.defs에서 홈디렉토리 생성 항목이 no로 되어 있을 시에 홈디렉토리 생성 위해서 사용하는 옵션
            - `-D` : default 확인 및 설정
        - 수정
            - `usermod option 계정명`
            - useradd에서 사용했던 옵션을 거의 다 사용할 수 있음.
            - usermod에서만 사용되는 옵션
                - `-a` : 보조그룹을 추가할 때 (-aG 형태로 쓰임)
                - `-m`(move) : 홈디렉토리 변경시에 실제로 디렉토리를 변경하기 위해서 사용(-md 형태로 쓰임)
                - `-l` : 사용자 계정명 변경
                - `-L` : 계정 lock(Lock걸리면 !붙음)
                - `-U` : 계정 lock 해제
        - 삭제
            - `userdel [option] 계정명`
            - `-r` : 삭제하고자 하는 계정에 홈디렉토리와 메일함 삭제시에 사용. 사용하지 않으면 홈디렉토리와 메일함이 남아있게 되서 동일한 UID로 배정받는 사용자가 사용할 수 있게 됨.
        - 실습
        
        ```powershell
        useradd -D -b /home -s /bin/bash
        
        사용자 생성 실습
        
        사용자   uid   gid          보조그룹     comment  home          shell   
        
        user01   1001  10                                 /home/user01  /bin/bash
        
        => useradd -g 10 user01
        
        user02   2000  2000          wheel        user    /home/user    /bin/sh
        
        => useradd -u 2000 -G wheel  -c "user"  -d /home/user -s /bin/sh user02 
        
        - 존재하지 않는 gid는 설정할 수 없고 uid와 같은 gid는 login.defs 파일에 의해서 지정하지 않아도 자동으로 부여됨
        
        user03   3000  1000                               /home/user03  /bin/sh
        
        => useradd -u 3000 -g 1000 -s /bin/sh user03
        
        user04   4000  4000          10	          guser   /home/user04  /bin/bash
        
        => useradd -u 4000 -G 10 -c "guser" user04
        
        - 존재하지 않는 gid는 설정할 수 없고 uid와 같은 gid는 login.defs 파일에 의해서 지정하지 않아도 자동으로 부여됨
        
        default 값 수정
        
        /bin/bash => /bin/sh 로 수정 => vi 편집기로
        
        vim /etc/default/useradd
        
        /bin/bash => /bin/sh
        
        :wq
        
        /home    => /home1 로 수정 => 명령어를 통해서 
        
        mkdir /home1
        useradd -D -b /home1
        
        testuser01 생성해서 확인해볼 것
        
        useradd testuser01 
        tail -1 /etc/passwd
        
        사용자 수정 실습
        
        사용자   uid   gid          보조그룹     comment  home          shell   
        user01   1001  1000         wheel                 /home/user01  /bin/bash
        
        => usermod -g 1000 user01
        
        user02   1002  2000         wheel,adm    user02   /home/user02  /bin/bash
        
        => usermod -u 1002 -aG adm -c "user02" -md /home/user02 -s /bin/bash user02
        
        - 보조그룹을 교체가 아니라 추가시에는 -aG 옵션을 부여해야 함
        
        user03   1003  1000         10                    /home/user03  /bin/bash
        
        => usermod -u 1003 -G 10 -s /bin/bash user03
        
        user04  1004  4000 10, 2000   guser    /home/user04 /bin/bash
        
        => usermod -u 1004 -aG 2000 user04
        
        testuser01 수정
        testuer01 => user05  
        /home1/testuer01 => /home/user05
        /bin/sh => /bin/bash
        
        usermod -md /home/user05 -s /bin/bash -l user05 testuser01
        ```
        
    - 그룹 관리
        - 생성
            - `groupadd [option] 그룹명`
            - `-g` : GID 설정
            - `-o` : 기존 GID 사용(-og형태로 사용)
        - 수정
            - `groupmod option 그룹명`
            - `-g` : GID 변경
            - `-o` : 기존 GID 사용
            - `-n` : 그룹명 변경
        - 삭제
            - `groupdel 그룹명`
        - 실습
        
        ```powershell
        group 실습
        
        id user01 => uid=1001(user01) gid=1001(user01)
        
        => groupadd -g 1001 user01
        => usermod -g 1001 user01
        
        id user02 => uid=1002(user02) gid=1002(user02)
        
        => groupmod -g 1002 user02
        
        id user03 => uid=1003(user03) gid=1003(user03)
        
        => groupadd -g 1003 user03
        => usermod -g 1003 user03
        
        id user04 => uid=1004(user04) gid=1004(user04)
        
        => groupmod -g 1004 user04
        ```
        
    - 암호
        - `passwd` 명령어를 통해서 패스워드 변경 가능
        - 암호 속성 변경
            - `chage option 계정명`
            - `-l`(소문자 엘) : 암호 속성 확인
            - `-m` : min date 변경
            - `-M` : Max date 변경
            - `-W` : warning date 변경
            - `-I`(대문자 아이) : inactive 변경
            - `-E` : expire 변경
            - `-d` : 암호 변경 시기 설정
        - 실습
        
        ```powershell
        user01 의 
        minum date 10
        maximum date 100
        warning date 10
        
        : chage -m 10 -M 100 -W 10 user01
        
        user02이 접속할때 암호변경을 하도록 설정
        
        : chage -d 0 user02
        
        user01 의 홈디렉토리를 삭제하고 다시 만들어 보시오.
        
        : rm -rf ~user01/
        
        cp -r /etc/skel /home/user01
        chmod 700 /home/user01
        chown -R user01:user01 /home/user01
        ```
