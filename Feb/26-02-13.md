## 📝 오늘 배운 내용 요약

1. NetworkManager
    - 모든 인터페이스 확인
        - `ip addr`
        - `ifconfig`
    - 특정 인터페이스 확인
        - `ip addr show [인터페이스명]`
        - `ifconfig [인터페이스명]`
    - 네트워크 경로 확인
        - `traceroute 특정 IP`
        - `tracepath 특정 IP`
    - nmcli 명령어
        - `nmcli device` : device 종류를 볼 수 있음.
        - `nmcli connection [sub-comand]`
            - add         : connection 생성
            - modify(mod) : connection 수정
            - delete(del) : connection 삭제
            - up          : connection 활성화
            - down        : connection 비활성화
            - reload      : connection 수정 사항 재 설정
            - show        : connection 설정 확인
        - 생성
            - `nmcli con add option argument`
            - option
                - 유동IP & 고정IP 공통설정
                    - `con-name ([connection.id](http://connection.id/))` : connection 이름
                    - `ifname (connection.interface)` : connection과 연결할 device
                    - `type (connection.type)`      : 통신방식 (ethernet, wifi, bluetooth,..)
                    - `autoconnect (connection.autoconnect)` : 부팅시 자동 연결(yes or no)
                - 고정IP 설정
                    - `ip4 (ipv4.addresses)`  :  고정IP 주소
                    - `gw4 (ipv4.gateway)`    :  게이트웨이 주소
            - (주의) ipv4.address 항목을 이용할 경우에는 ipv4.method  manual을 반드시 기입
            - 예제
            
            ```powershell
            장치명 : enp0s7
            통신방식 : 유선
            유동IP 방식(dhcp)
            부팅시 자동 연결
            
            nmcli con add con-name dhcp7       |   connection.id  dchp7
                          ifname enp0s7        |   connection.interface enp0s7
                          type ethernet        |   connection.type  802-3-ethernet
                          autoconnect yes      |   connection.autoconnect yes
            
            nmcli con
            
            nmcli con show dhcp7
            
            ipv4.method      auto   (자동)
            
            --------------------------------------------------------------------------------------------------------------
            
            장치명 : enp0s7
            통신방식 : 유선
            고정IP 방식 : 10.100.5.100/24
                          10.100.5.2
            부팅시 자동 X
            
            nmcli con add con-name static7     |   connection.id  static7
                          ifname enp0s7        |   connection.interface enp0s7
                          type ethernet        |   connection.type  802-3-ethernet
                          autoconnect yes      |   connection.autoconnect no
                          ip4 10.100.5.100/24  |   ipv4.addresses 10.100.5.100/24
                          gw4 10.100.5.2       |   ipv4.gateway   10.100.5.2
                                               |   ipv4.method    manual
            
            nmcli con 
            
            nmcli con show static7
            
            connection.autoconnect no         (autoconnect yes)
            
            ipv4. method           manual  (수동)
            
            ipv4.addresses         10.100.5.100/24
            
            ipv4.gateway           10.100.5.2
            ```
            
        - 수정
            - `nmcli con modify (mod) option argument`
            - option
                - nmcli con show 연결명을 실행 했을 때 나오는 항목이 모두 옵션
                - `connection.id [변경할 연결이름]`
                - `ipv4.address [변경할 IP 주소]`
                - `ipv4.dns [dns주소]`
                - `ipv4.gateway [변경할 주소]`
                - `ipv4.method [auto or manual]`
                - `+ipv4.addressess [추가할 ip 주소]`
                - `-ipv4.addressess [제거할 ip 주소]`
            - 예제
            
            ```powershell
            기존 연결 dhcp7 에 대해서 아래 와 같이 수정을 한다면
            
            연결 이름 dhcp7  static7-1
            유동IP => 고정IP
            ip 주소 : 10.100.5.200/24
            dns 주소: 10.100.5.100
            
            nmcli con mod dhcp7 connection.id static7-1 
                                ipv4.method manual 
                                ipv4.addresses 10.100.5.200/24 
                                ipv4.gateway 10.100.5.2
                                ipv4.dns 10.100.5.100
            
            연결명 수정 static7 => dhcp7-1
            고정IP => 유동IP
            dns 주소 8.8.8.8 10.100.5.100
            자동연결 O
            
            nmcli con mod static7 connection.id  dhcp7-1
                                  ipv4.method    auto
                                  ipv4.addresses ""
                                  ipv4.gateway   ""
                                  connection.autoconnect  yes
                                  ipv4.dns 8.8.8.8
                            
                                  
            [수업시간에 했던 코드]                      
            **ex) nmcli con mod dhcp1 connection.id static2 ipv4.method manual ipv4.addresses 192.168.100.60/24 
            ipv4.gateway 192.168.100.2  +ipv4.addresses 192.168.100.70/24  ipv4.dns 8.8.8.8**
            ```
            
        - 인터페이스 관리
            - 연결하기
                - `nmcli device connect eno~~~`
            - 연결 끊기
                - `nmcli device disconnect eno~~~`
            - 상태 보기
                - `nmcli device status`
        - `nmtui`
            - 텍스트 기반 인터페이스
    - 호스트명 확인
        - `hostname`
        - `cat /etc/hostname`
    - 호스트명 변경
        - `hostnamectl hostname [호스트명]`
2. 네트워크 실습

```powershell
초기 설정으로 초기화

현재 IP 주소를  2가지 방법으로 확인해보시오.

전체 확인

ifconfig
ip addr

eth2 인터페이스 주소 확인

ifconfig eth2
ip addr show eth2

add로 연결 생성

dhcp => 유동IP

nmcli con add con-name dhcp type ethernet ifname eth2 

static1 => ip : 192.168.56.200/24, gw : 192.168.56.1 

nmcli con add con-name static2 type ethernet ifname eth2 ip4 192.168.56.200/24 gw4 192.168.56.1

modify 로 연결 수정

- dhcp 연결을 static 으로 이름 변경 후 
- 192.168.56.120으로 고정 아이피 설정
- 192.168.56.30 주소 추가
- 8.8.8.8 dns 설정 
- 자동 연결 안되도록 수정
- method manual 로 설정 후 네트워크 연결 확인

nmcli con mod dhcp connection.id static ipv4.addresses 192.168.56.120/24 ipv4.gateway 192.168.56.1 +ipv4.addresses 192.168.56.30/24 ipv4.dns 8.8.8.8 
connection.autoconnect no ipv4.method manual

192.168.56.120 주소를 제거하시오.

nmcli con mod static -ipv4.addresses 192.168.56.120/24

hostname을 nobreak.com로 설정

hostnamectl hostname nobreak.com
```

## * 지난 시간 배운 것에 대한 실습

```powershell
<repo 실습>

﻿스냅샷 초기화

rocky DVD 인식

DVD 마운트

********************************************

umount /dev/cdrom

mkdir /media/cdrom

mount /dev/cdrom /media/cdrom

*******************************************

기존 repo 파일을 새로 생성한 backup 디렉토리에 이동 시켜놓고

cd /etc/yum.repos.d/

mkdir backup

mv r* b*

dvd 와 인터넷 (BaseOS,AppStream) 을 통해서 다운을 받을 수 있도록 repo 파일 설정
인터넷 주소

- BaseOS : https://dl.rockylinux.org/pub/rocky/9/BaseOS/x86_64/os/
- AppStream : https://dl.rockylinux.org/pub/rocky/9/AppStream/x86_64/os/

로컬 주소 

- BaseOS : file:///media/cdrom/BaseOS/
- AppStream : file:///media/cdrom/AppStream/

++++++++++++++++++++++++++++++++++++++++++++++++++++++++

vi /etc/yum.repos.d/test.repo

[dvd-baseos]
name=DVD-BaseOS
baseurl=file:///media/cdrom/BaseOS
enabled=1
gpgcheck=0

[dvd-appstream]
name=DVD-AppStream
baseurl=file:///media/cdrom/AppStream
enabled=1
gpgcheck=0

[net-baseos]
name=Net-BaseOS
baseurl=https://dl.rockylinux.org/pub/rocky/9/BaseOS/x86_64/os/
enabled=1
gpgcheck=0

[net-appstream]
name=Net-AppStream
baseurl=https://dl.rockylinux.org/pub/rocky/9/AppStream/x86_64/os/
enabled=1
gpgcheck=0

:wq
++++++++++++++++++++++++++++++++++++++++++++++++++++++++

repository cache  값 재구성

dnf makecache

repo 활성화 상태 확인

dnf repolist all

zsh 패키지를 rpm 명령어를 통해서 설치

rpm -Uvh /media/cdrom/BaseOS/Packages/z/zsh-XXXXXXXX.rpm

rpm -qa | grep zsh

zsh 패키지를 rpm 명령어를 통해서 삭제

rpm -e zsh

rpm -qa | grep zsh

ksh 패키지를 yum 명령어를 통해서 설치(한번에 설치까지)

yum -y install ksh

rpm -qa | grep ksh

zsh 패키지를 yum 명령어를 통해서 설치(ISO 이미지의 .rpm 파일을 통해서)

yum -y localinstall /media/cdrom/BaseOS/Packages/z/zsh-xxxxxxxxx.rpm

ksh 패키지를 yum 명령어를 통해서 삭제(한번에 삭제)

yum -y remove ksh

그룹 패키지 리스트 확인

yum groups list

Security Tools 패키지 정보 확인

yum groups info "Security Tools" (한글버전이신 분들은 "보안 툴")

Security Tools 그룹 패키지 설치

yum groups install "Security Tools"  -y (한글버전이신 분들은 "보안 툴")

Security Tools 그룹 패키지 삭제

yum groups remove "Security Tools" -y (한글버전이신 분들은 "보안 툴")

/var/www 디렉토리가 어떤 패키지로 인해서 생성된 것인지 확인하시오.

rpm -qf /var/www          (- httpd 패키지가 설치 되어 있지 않으면 결과가 안나올 수 있음)

or

yum provides /var/www

repository 에서 http 관련 패키지 검색 

yum list | grep http

yum search http

httpd 패키지 설치

yum install httpd

해당 패키지로 인해서 생성된 설정 파일 확인

rpm -qc httpd

해당 패키지로 인해서 생성된 문서 파일 확인

rpm -qd httpd

해당 패키지에 대한 정보를 확인하시오.

rpm -qi httpd

yum info httpd

httpd 서비스 시작 및 서비스 활성화 (해당 서비스는 httpd.service)

systemctl enable httpd
systemctl start httpd

or

systemctl enable httpd  --now

서비스 종료 및 비활성화

systemctl disable httpd
systemctl stop httpd

or

systemctl disable httpd --now

해당 패키지 삭제

yum remove httpd

다시 설치 (설치 질문 없이 설치)

yum -y install httpd
```
