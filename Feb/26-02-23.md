## 📝 오늘 배운 내용 요약

1. DNS
    - IP 주소를 Domain 형식으로 변환해주는 서비스
    - Domain name
        - 도메인 이름은 영문자(a-z), 숫자(0-9) 10개, 특수기호(-) 1를 합쳐 총 37개의 글자 조합으로 나타낼 수 있음
        - 영문자의 경우 대문자 소문자의 구별은 하지 않으며 특수 기호인(-)은 도메인의 처음과 끝에 올 수 없음
        - 도메인 이름은 전 세계적으로 겹치지 않게 하나만 부여되며 First Come, First Served 원칙에 따라 부여
        - 도메인 이름의 체계는 피라미드 구조를 하고 있으며, 각 단계별로 독자적으로 관리
            - Domain Name 구조
                - 루트
                    - .
                - 1단계 Top Level Domain
                    - 국가 도메인(ccTLD) : kr, us, jp, cn, uk, fr
                    - 일반 도메인(gTLP) : com, org, edu
                - 2단계 도메인
                    - 조직의 속성을 구분하는 도메인
                    - co(영리), or(비영리), go(정부), ac(대학/대학원), mil(국방조직)
                    - 일반 도메인은 Second Level이 없음
                - 3단계 도메인
                    - 조직이나 서비스의 이름을 나타냄
                    - naver, goolge, nate, kernerl, nobreak
                - Subdomain
                    - 호스트가 위치
                    - www, blog, cafe, news
            - 도메인을 표기할 때는 낮은 단계부터 표현하여 최상위 도메인이 가장 뒤에 나타남

    - DNS 관련 파일
        - `/etc/hosts`
            - 인터넷 주소와 호스트 이름의 관계를 설정
            - 매우 간단하게 설정할 수 있지만 이 파일의 내용은 로컬 시스템에서만 유효하게 사용 가능
        - `/etc/resolv.conf`
            - 호스트가 사용할 DNS 서버의 주소를 설정
            - 자신의 호스트가 DNS 서버라면 자신의 IP 주소를 적어줘야 함
2. Domain 동작 방식
    - 재귀 쿼리
        - 사용자는 자신의 요청한 DNS 서버에게만 요청을 전송하고, 요청에 대한 응답을 수신함
        - 재귀 쿼리는 일반적으로 클라이언트가 DNS 서버에게 요청할 때 발생
        - 경우에 따라 DNS가 다시 다른 DNS 서버에게 요청하는 경우에도 사용
    - 순환 쿼리
        - 클라이언트로부터 받은 이름 요청에 대한 답변이 없는 경우 다른 DNS 서버로부터 단계적으로 질의하는 과정을 수행
3. 조회 방식
    - 정방향 조회
        - 도메인 주소를 요청하면 IP주소로 응답
        - `host 도메인 [네임서버] [option]`
            - ex) `host naver.com 8.8.8.8` , `host naver.com`
            - -t : 레코드 종류 지정
            - -a : 모든 종류의 레코드 조회
            - -l : 리스트 모드로 결과 출력
            - -v : 상세 정보 출력
        - `nslookup 도메인 [네임서버]` or `nslookup`
            - nslookup 프롬프트가 나옴
    - 역방향 조회
        - IP 주소를 요청하면 도메인 주소로 응답
        - DNS 서버는 IP 주소가 아닌 도메인 주소를 저장하기 때문에 도메인 주소로 IP 주소를 찾는 건 힘들다
        - in-addr.arpa
4. DNS 캐시
    - 한번 찾은 도메인은 엔드포인트와 DNS 서버 둘 다에서 저장함

5. DNS 레코드 종류
    - A & AAAA
        - 구매한 도메인이 가리키는 IP 주소를 설정하는 레코드
        - IPv4는 A, IPv6는 AAAA를 사용
        - 이 레코드를 설정함으로써 인터넷 사용자가 웹 브라우저에 도메인 입력 시 설정된 IP 주소로 연결됨
    - CNAME
        - 다른 DNS 레코드의 별칭 레코드
        - Canonical Name의 약자로 주 도메인명 외에 보조 도메인명을 지정하는 레코드
        - 예시) hydradns.com을 요청할 때와 보조 도메인명으로 지정한 www.hydradns.com을 요청할 때 모두 동일한 IP 주소로 연결됨
    - MX
        - Mail Exchanger 레코드를 의미
        - MX 레코드가 DNS에 설정되어 있어야 도메인과 메일 서버가 연결되어 [xxx@hydradns.com](mailto:xxx@hydradns.com) 형식의 이메일 주소 이용이 가능하게 됨
    - NS
        - Name Server 레코드를 의미하며, 어떤 DNS 서버가 해당 도메인에 대한 권리를 가지고 있는지를 가리키는 레코드
        - 도메인 별로 주 네임서버와 백업용 네임서버들을 지정하기도 함
    - PTR
        - Pointer Record를 의미하며, 역방향 DNS 조회라고도 함
        - A 레코드와 반대로 특정 IP 주소에 해당하는 도메인의 이름 정보를 저장
        - 도메인이 아닌 IP를 질의하여 도메인을 확인하는 방식이기 때문에 전송이 아닌 수신하는 쪽에서 메일발송 서버의 IP를 조회하여 해당 도메인이 등록된 PTR 레코드 값과 일치하면 정상메일로 판단하고 메일을 수신하는 역방향 질의 방식으로 진행
        - 최근에는 PTR 레코드가 없는 발송 서버 IP에서 보내는 메일은 차단하거나 스팸 처리하는 경우가 많음
    - SOA
        - 영역의 권한 시작 정보
        - 영역을 구성하는 가장 기본 레코드
        - 모든 영역은 한 개의 SOA 레코드를 가지고 있음
        - SOA 레코드는 마스터-슬레이브 간의 Zone Transfer에 대한 설정이 주 내용임을 알 수 있음
        - 상세한 설명
            - Zone 파일은 SOA 레코드로 시작
            - Refresh : Secondary 서버가 Primary 서버에 SOA 레코드가 업데이트 되었는지 질의하기 위해 기다리는 시간
            - Retry : Primary 서버가 응답하지 않는 경우 다시 질의하기까지 기다리는 시간
            - Expire : 만약 Secondary 서버가 이 시간 만큼 Primary 서버로부터 응답을 받지 못하면 해당 Zone에 대한 쿼리에 응답을 중지해야 함
            
            ```powershell
            $TTL 3H
            @   3600     IN  SOA     ns.icann.org. noc.dns.icann.org. (
            							        2020080302  ;Serial (Serial 번호 슬레이브 - 마스터 zone 파일 비교)
            							        7200        ;Refresh (주기(초) 슬레이브 -> 마스터 체크 주기)
            							        3600        ;Retry (슬레이브 -> 마스터 접속시 얼마후 재접속할지)
            							        1209600     ;Expire ( Zone 파일의 유효 기간)
            							        3600        ;Minimum TTL ( zone 파일을 읽은 네임서버에 얼마동안 보관할지 )
            								)
            ```
            
    - TXT
        - 관리자가 DNS 레코드에 도메인에 대한 텍스트 정보를 입력해 넣을 수 있도록 해주는 레코드
    - URL
        - 호스트네임(hostname)을 다른 URL로 다시 전송하는 레코드
        - 예시) [www.example.com](http://www.example.com/) 도메인을 example.com으로 전송되도록 함
    - Framed URL
        - URL 레코드와 비슷하나 웹페이지로 클라이언트를 연결하는 대신 웹 페이지가 웹 서버의 프레임(frame)에 보이도록 하는 레코드
        - 예시) 페이지로 연결하는 실제 URL이 아님에도 클라이언트의 브라우저가 해당 도메인명(예: [www.example.com](http://www.example.com/))을 보여줌
6. DNS 영역
    - Zone 파일은 DNS의 마스터 - 슬레이브 설정 시, 슬레이브가 DNS 설정 복제(replication)을 위해 마스터에서 주기적으로 업데이트 여부를 확인하고, 업데이트가 있다면 가져오는 정보이기도 함
    - Zone 파일은 결국 DNS Zone에 대해 SOA(Start of Authority) 레코드와 해당 zone에 포함된 서브 도메인들의 Resource 레코드 값을 정의한 텍스트 파일
    - 특정 도메인 범위에 대한 관리를 담당할 수 있음
    - `/var/named/` 안에 `*.zone` 으로 생성
    - `/var/named/named.empty` 파일을 템플릿으로 사용함
