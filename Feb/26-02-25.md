1. HTTPS (HTTp Secure)
    - HTTP의 암호화된 버전
    - SSL (Secure Socket Layer) - 인터넷 보안 프로토콜
    - TSL (Transport Layer Security) - SSL이 표준화 되면서 바뀐 이름
    - HTTPS 암호화 방식
        1. 클라이언트 → 서버로 랜덤 데이터와 사용 가능한 암호화 방식을 보낸다.
        2. 서버 → 클라이언트로 랜덤 데이터, 사용할 암호화 방식과 SSL 인증서를 보낸다.
        3. 클라이언트는 서버에게 받은 인증서의 CA가 자신이 들고 있는 CA 리스트에 있는지 확인하고, 있다면 CA의 공개키로 복호화 한다. 이는 곧 CA 비밀키에 의해 암호화됐다는 것이므로 인증서의 신원을 보증해준다. (공개키 암호화 방식)
        4. 클라이언트는 자기가 보낸 랜덤 데이터와 서버로부터 받은 랜덤 데이터를 조합하여 임시 키(pre master secret key)를 만든다.
        5. 만들어진 임시 키를 인증서의 공개키로 암호화하여 서버에게 보낸다.
        6. 서버는 자신이 들고 있던 비밀키로 임시 키를 복호화 한다.
        7. 이로써 클라이언트와 서버는 동일한 임시 키를 공유하게 되는데, 일련의 과정을 거쳐 master secret 값을 만들고 세션 키를 생성한다.
        8. 이렇게 만들어진 세션 키로 암호화된 데이터를 주고받는다. (대칭키 암호화 방식)
        9. 세션이 종료되면 클라이언트와 서버 모두 세션키를 폐기한다.

2. 인증서 발급 과정
    - 개인키 생성
        - `openssl genrsa -out (key파일명) (key길이)`
        - key_length - 512, 768, 1024, 2048
    - CSR 생성
        - 인증서를 발급 받기 위해 필요한 서명
        - `openssl req -new -key (key파일명) -out (csr파일명)`
    - CRT 생성(자필 서명)
        - `openssl x509 -req -signkey (key파일명) -in (csr파일명) -out (crt파일명)`
3. HTTPS 설정
    - key 파일은 `/etc/pki/tls/private/` 안에 저장
    - crt 파일은 `/etc/pki/tls/certs/` 안에 저장
    - `mod_ssl` 설치 : SSL/TLS를 지원하는 모듈
    - `/etc/httpd/conf.d/ssl.conf` 파일 수정
        - SSLCertficateFile (crt 파일 경로)
        - SSLCertificateKeyFile (key 파일 경로)
        - SSLProtocol - [https://ssl-config.mozilla.org](https://ssl-config.mozilla.org/)에서 버전 확인 : `httpd -v` `openssl -v`
        - SSLCipherSuite - [https://ssl-config.mozilla.org](https://ssl-config.mozilla.org/)에서 버전 확인 : `httpd -v` `openssl -v`
    
    ```powershell
    cp 00-vhost-default.conf 05-vhost-https.conf
    vi 05-vhost-https.conf
    
    -----------------------------------------------------------------
    
    <Virtualhost *:443>
            DocumentRoot "/var/www/html-https"
            ServerName web.est.main
            SSLEngine on
            SSLCertificateFile /etc/pki/tls/certs/est.crt
            SSLCertificateKeyFile /etc/pki/tls/private/est.key
            SSLProtocol -all +TLSv1.2 +TLSv1.3
            SSLCipherSuite ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384:DHE-RSA-CHACHA20-POLY1305
    </Virtualhost>
    ```
    
4. HTTPS 리다이렉션
    - `conf` 기본 파일 수정
    - RewriteCond, RewriteRule →  [https://ssl-config.mozilla.org](https://ssl-config.mozilla.org/) 에서 확인 후 변경 가능
    
    ```powershell
    vi 00-vhost-default.conf
    
    -----------------------------------------------------------------
    
    <Virtualhost *:80>
            DocumentRoot "/var/www/html-default"
            ServerName web.est.main
            RewriteEngine On
            RewriteCond %{REQUEST_URI} !^/.well-known/acme-challenge/
            RewriteRule ^.*$ https://%{HTTP_HOST}%{REQUEST_URI} [R=301,QSA,L]
    </Virtualhost>
    
    <Directory /var/www/html-default>
            AllowOverride none
            Require all granted
    </Directory>
    ```
    
5. 역방향 프록시(Reverse Proxy)
    - `yum -y install haproxy` :  로드 밸런싱 특화 도구
    - `vi /etc/haproxy/haproxy.cfg`
    - 역방향 프록시 HTTPS 설정
        - 인증서 발급
            - `openssl x509 -req -signkey ha.key -in ha.csr -out ha.crt` -   자체 서명 인증서
        - `key파일`과 `crt파일`을 하나로 합쳐서 `/etc/haproxy/certs/` 안에 저장
            - `cat (key파일) (crt파일) > /etc/haproxy/certs/(이름).pem`
            
            ```powershell
            mkdir /etc/haproxy/certs/
            cat ha.key ha.crt > /etc/haproxy/certs/ha.pem
            ```
            
        - `/etc/haproxy/haproxy.cfg` 파일 수정
        
        ```powershell
        vi /etc/haproxy/haproxy.cfg
        
        +++++++++++++++++++++++++++++++++++++++++++++++++++
        
        frontend main
            bind *:443  ssl crt /etc/haproxy/certs/ha.pem ssl-min-ver TLSv1.2
            acl url_static       path_beg       -i /static /images /javascript /stylesheets
            acl url_static       path_end       -i .jpg .gif .png .css .js
        
            acl https_request ssl_fc
            http-request redirect scheme https code 301 if !https_request
        
            use_backend static          if url_static
            default_backend             web
        
        +++++++++++++++++++++++++++++++++++++++++++++++++++
        ```
